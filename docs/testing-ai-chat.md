# AIチャットのテスト手順

## 1. API キーの準備

### 方法A: .env ファイルを使う（ローカル推奨）

1. プロジェクトルート（`wp/`）に `.env` を作成する。
2. 次の1行を追加し、`sk-...` の部分をあなたの OpenAI API キーに置き換える。

   ```
   OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxx
   ```

3. `.env` は **Git にコミットしない**こと。リポジトリに `.gitignore` がある場合は `.env` を追加する。

   ```bash
   echo ".env" >> .gitignore
   ```

4. 初回は `.env.example` をコピーして編集してもよい。

   ```bash
   cp .env.example .env
   # .env を開いて OPENAI_API_KEY= の右にキーを貼り付ける
   ```

### 方法B: 環境変数を直接渡す

ターミナルでサーバーを起動する前に、同じシェルで:

```bash
export OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxx
php -S localhost:8000
```

※ このターミナルを閉じると変数は消えます。

---

## 2. サーバーを起動する

プロジェクトルート（`wp/`）で:

```bash
php -S localhost:8000
```

- PHP ビルトインサーバーでは `.htaccess` のリライトは効かないため、URL は `http://localhost:8000/?page=about` のようなクエリ形式になる。
- Apache で動かしている場合は、そのドメイン・パスのまま次の手順を行う。

---

## 3. ブラウザでサイトを開く

1. ブラウザで `http://localhost:8000/`（またはあなたの開発用URL）を開く。
2. 画面右下に **AIチャット** のボタン（吹き出しアイコンなど）が出ていることを確認する。
3. そのボタンをクリックしてチャットパネルを開く。

---

## 4. メッセージを送ってテストする

1. 入力欄に短い質問を入力する。例:
   - 「会社の住所を教えてください」
   - 「Webサイト制作について知りたい」
   - 「お問い合わせはどうすればいいですか」
2. 送信ボタンを押す。
3. **正常時**: 数秒以内に AI の返答が表示される。
4. **API キー未設定時**: 「申し訳ございません。一時的に応答できません。しばらくしてからお試しください。」のようなメッセージが表示される。→ 手順1を確認する。
5. **ネットワークエラー時**: 「エラーが発生しました。もう一度お試しください。」などが表示される。→ コンソール（F12 → Network）で `/api/chat.php` のレスポンスを確認する。

---

## 5. 動作確認のポイント

| 確認項目 | 期待される結果 |
|----------|----------------|
| チャットボタンが表示される | 画面右下に表示される |
| パネルが開く | クリックで開閉する |
| 送信すると「入力中」表示 | タイピングインジケーターが出る |
| 返答が表示される | 日本語で簡潔な回答が返る |
| キー未設定で送信 | 上記の「一時的に応答できません」メッセージ |

---

## 6. トラブルシューティング

### 「API key not configured」や 500 エラー

- `.env` の `OPENAI_API_KEY=` の右にキーが入っているか、余計なスペースや改行がないか確認する。
- 環境変数で渡している場合は、**同じターミナル**で `echo $OPENAI_API_KEY` を実行し、先頭が `sk-` で表示されるか確認する。

### `/api/chat.php` に 404 が出る

- PHP ビルトインサーバーなら、ルートが `wp/` になっているか確認する。`wp/` で `php -S localhost:8000` を実行していれば `http://localhost:8000/api/chat.php` でアクセスできる。
- Apache の場合は、`api/chat.php` が実在するパスか、リライトで隠れていないか確認する。

### CORS エラー

- チャットは同じオリジン（同じドメイン・ポート）から開いている想定。`/api/chat.php` への fetch は同じサイトなので、通常は CORS は発生しない。別ドメインからテストしている場合は、同じオリジンで開き直す。

---

## 7. API を直接叩いて試す（任意）

ターミナルで:

```bash
curl -X POST http://localhost:8000/api/chat.php \
  -H "Content-Type: application/json" \
  -d '{"message":"会社の住所を教えてください"}'
```

- 成功時: `{"reply":"..."}` のような JSON が返る。
- キー未設定時: `{"error":"...", "reply":"申し訳ございません..."}` が返る。

これで「API キー」「サーバー」「フロント」のどれに問題があるか切り分けやすい。
