## 主な成果

バックオフィス業務を柿崎が担当し、代表の渡沢から支払い処理、入金確認、納税対応などの定常業務を引き継ぎ、月40-50分程度の時間削減を実現した 。ナインビルのオフィス改善では大量の不用品を処理し、ラインスペースを確保して面接や来客対応が可能な環境を整備した 。採用活動では2月から炎天職を利用してウェブプログラマーとウェブディレクターの採用を進め、550件のスカウトを送信し16件の応募を獲得したが、内定辞退が1件あり現時点で確定ゼロの状況 。
## バックオフィス業務の引き継ぎと体制整備

### 代表からの業務移管

柿崎がバックオフィス担当として、渡沢が従来行っていた以下の業務を引き継いだ :
- **支払い処理と振込業務**: ネットバンキングへのアクセス権限を取得し、毎月約35件の支払い処理を実施 
- **入金確認と督促**: 決算時に税理士から指摘された入金漏れや不一致をリスト化し、各担当者に顧客への連絡を依頼 
- **納税対応**: 法人税などの納税時期に税理士からの指示に基づき、ネットバンキングで入金処理を実施 

### システム環境の整備

**ネットバンキングの再構築**:
- 従来は渡沢のPCのWindows版からのみアクセス可能だった状態を、Mac版に契約変更 
- 管理者を2人体制に再セッティング 
- ログイン情報が不明だった別の銀行口座については、銀行に出向いて再発行手続きを実施 

**経費精算システムの導入効果**:
渡沢が作成した経費精算システムのプログラム(ChatGPT活用)により、柿崎の毎回の計算・集計作業が不要になり、時間削減を実現した 
### 定常業務の範囲

**月次業務** :
- 経費支払い処理: 約35件/月
- 勤怠管理チェック
- 有給管理
- ご飯給管理と発注

**年次・スポット業務** :
- 入金確認: 約10件/年(決算時に課題が発覚)
- 納税処理: 年数回
- 健康診断の調整と支払い
- 退職・入社手続き(都度)
- サービス導入検討と情報収集
- 名刺関連業務(必要時)

## オフィス環境の改善

### ナインビルの大規模整理

**不用品の処分**:
- 使用不可のモニター、不要な木材など大量の不用品をトラックに積載できる量まで処分 
- ミクラとサテライトの不用品も内に集約して一括処分 
- オレンジ色の袋に二重梱包し、とがったものによる傷防止対策を実施 

**オーナー対応の課題**:
- オーナーから「事務所でゴミを集めるのはいかがなものか」とクレームを受ける 
- 不動産会社に連絡して説明し、完全養生での対応を約束 
- 当日は入口から階段、部屋までの全経路を養生シートで保護して作業を実施 

**改善効果**:
- ラインスペースを確保し、面接や来客対応が可能な環境に改善 
- PMCとの打ち合わせなどにも対応可能になった 

### 今後の環境整備計画

渡沢の説明によると、以下の工事・改修を予定している :
- サテライトの共有部分の工事による新しい状態への改修
- ナインビルの工事実施(人を迎える場所としての整備)
- メンバーがオフィスを使いやすく、出社して気分よく仕事ができる環境づくり

## 採用活動の実施状況

### 採用の目的と背景

**リソース不足の課題** :
- ゲーム事業や他サービス、販売代理店事業への注力リソースが不足
- 受託プロジェクトが多く、制作部門とPMチームの負担が大きい
- 渡沢が制作業務に追われ、経営的業務や上流業務が逼迫している状態

**募集職種** :
- **ウェブプログラマー**(制作部門): 2年以上の経験必須(一から教える余裕がないため)
- **ウェブディレクター**(PMチーム): 経験者が望ましいが、プロジェクト管理経験やストレス耐性、人物面での適性も重視

### 炎天職を利用したスカウト採用

**採用手法の変更**:
従来の求人票公開・応募待ちから、炎天職の求職者リストを閲覧してこちらから理想的な人材にスカウトを送る方式に変更した 
**母数の課題** :
- 炎天職から「取れます」と説明を受けて契約したが、実際には希望条件の母数が限定的だった
- ディレクターで2年以上経験者に絞ると母数が少ない
- プログラマーも年収600-800万円帯で絞り込んでも期待ほど母数が広がらなかった

**スカウト実績** :
- 送信可能なスカウト上限: 540件(当初470件から炎天職のミスで増加)
- 追加メッセージ2回送信可能のため、合計約1,500回以上のアプローチを実施
- ディレクター: 約300件アプローチ
- プログラマー: 約200件アプローチ(残り35件を24日に再送予定)

**応募率**:
3.2%(炎天職からは同業種平均2.5%より高いと説明されたが、同一条件での比較ではないため参考程度) 
### 面接プロセスと結果

**面接体制** :
- 一次面接: 柿崎と宇住(30分、Web実施)
- 二次面接: 柿崎と渡沢(1時間)
- 日程調整や合否連絡は基本的に宇住が担当

**選考結果** :
- 応募: 16件
- 連絡途絶: 2件
- 面接実施可能: 14件
- 一次面接実施: 10件
- 二次面接進出: 4件
- 内定出し: 1件(プログラマー、東京外在住のため引っ越し困難で辞退)
- お祈り: 8件
- 連絡取れず: 3件
- 面接予定: 2件(ディレクター枠)
- **確定: 0件**

### 求人票の充実化

**作成内容** :
- 1,500文字程度のフル活用した詳細な求人票
- 写真の追加
- 会社紹介動画の制作(渡沢が写真から作成、スタッフの動きも加工) 
- メンバー情報の追加(内田、宇住、柿崎、渡沢)
- 炎天職運営の口コミサイトへの口コミ投稿依頼 

### 炎天職のサポート評価

**サポートの限界** :
- 求人票の改善アドバイスは統計に基づいて提供される
- スカウト半分投げても応募ゼロの状況に対しては「頑張って何回も送ることが大事」という回答のみ
- 根本的・画期的なサポートは提供されない印象

**求人内容の認識ずれ**:
求人票に完全リモートと記載していないにもかかわらず、応募者の一部が完全リモート・在宅勤務を希望するケースがあった 
## 今後の対応と課題

### 採用活動の継続方針

**残りスカウトの活用** :
- 残り35件のスカウトは全てプログラマーに集中投下(ディレクターより優先度が高いため)
- 条件を広げて不適合な人材に送るより、理想的な人材に再度スカウトを送る戦略
- 24日に再送予定

**採用可能性** :
- プログラマー: 0人または奇跡的に複数応募があれば1人取れる可能性
- ディレクター: 面接予定の2人次第(どちらもダメなら0件)

**炎天職への対応**:
採用が取れなかった場合、炎天職に何らかの交渉が必要になる可能性がある 
### バックオフィス業務の課題

**勤怠管理の徹底** :
- 現状: 毎日入力されていない状況
- 目標: 毎日の開始時と終了時に入力するルールの徹底
- 課題: お願いするだけでは実施されない可能性が高く、実施方法の検討が必要

**入金管理の改善** :
- 現状: 年1回の決算時にまとめて確認(最長1年前の入金漏れを指摘)
- 提案: 毎月1回、各担当者に5-10分程度の時間を確保してもらい、以下を確認
    - 前々月の入金が正しく入っているか
    - 今月出す請求書リストが正しく作成されているか

**請求書管理の精度向上** :
- 決算時に発覚した問題: 請求書の重複発行、破棄依頼した請求書がシステム上でカウントされている
- 必要性: お金周りの管理は完璧を目指し、毎月の請求書一覧を綺麗に保つ
- 多発はしていないが、事後チェックが大変になるため予防が重要

### 渡沢の方針と今後の展開

**バックオフィスの存在意義** :
- メンバーが快適に仕事ができる環境を作ることがポイント
- みんなから感謝される側の作業を目指す
- 一般的な会社のルールに縛られて働きにくくなるのは避けたい

**業務効率化の方向性** :
- 勤怠入力が面倒なら自動カウントの仕組みを検討
- 請求周りは銀行データから自動突き合わせを検討
- できるだけメンバーの手間にならない方針

**柿崎の役割の将来像** :
- 定常業務まで落とせたら柿崎から剥がす予定
- 柿崎がバックオフィスに特化したいわけではないと認識
- よりクリエイティブな方向にも進んでほしい
- 現状は役割的にちょうど良いので担当してもらっている状態

**採用に対する基本姿勢** :
- 人が入らなくても一応成立している状態
- お金はもったいないが、無理やり誰かを入れる判断はしない
- PMチームも「誰でもいいから欲しい」状態ではない
- 将来的に未来が変わるような人をしっかり選びたい

**採用要件の変化** :
- 当初: 全てを理解し技術責任者になれるような経験豊富な人材を想定
- AIコーディングの進展により: 渡沢の指示をそのまま引き継いでくれる地道に作業できる人材へシフト
- 内定を出した人(辞退)も地道にやるタイプだった
- ディレクターは学習が必要な人が入ると内田の負担が増えるため、ハードルを上げて選考

## アクションアイテム

- 柿崎: 残り35件のスカウトを24日にプログラマーに再送 
- 柿崎・渡沢: 炎天職で採用が取れなかった場合の次の採用戦略を検討 
- 柿崎: 勤怠の毎日入力を実現する具体的な方法を検討 
- 柿崎: 毎月の入金確認・請求書リスト確認の仕組みを各担当者と調整 
- 柿崎・渡沢: 請求書管理の自動突き合わせシステムを検討 
- 渡沢: ナインビルとサテライトの工事・改修を進める
